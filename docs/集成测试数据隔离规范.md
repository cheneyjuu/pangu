# 集成测试数据隔离规范

> 关联业务：Pangu 基于真实 PostgreSQL、Flyway 最终状态和登录会话的集成测试。

## 基本约束

1. 测试清理数据时按外键依赖从子表到父表执行；新增会话、审计或凭证表后，同步检查所有删除账号的测试夹具。
2. 断言数据库种子时以全部 Flyway 迁移执行后的最终状态为准，不能固定依赖某个中间迁移暂时创建的菜单或权限。
3. 历史迁移不得为修测试而改写；最终状态发生合法演进时，更新测试断言，必要时通过新迁移修正生产数据。
4. 测试失败后先区分业务回归、夹具清理遗漏和历史种子被后续迁移取代，不能通过弱化业务断言掩盖真实问题。

## 2026-07-15 错误复盘

- `V3.80` 新增刷新会话外键后，房产绑定测试仍直接删除账号，导致清理阶段违反 `t_auth_refresh_session.account_id` 外键。
- 菜单测试仍假定 `V3.45` 创建的 `community-space` 存在，但 `V3.77` 已将同一菜单 ID 用于最终的 `property-service-organization`。
- `V3.85` 的结算项、付款凭证等子表同时引用项目工程项或项目附件；集成测试只删除项目根时，多个级联分支与 `NO ACTION` 外键相遇，连续在结算项、付款凭证处清理失败。
- 刷新会话外键问题不只影响房产绑定测试：任何执行登录后再删除临时账号的集成测试都会签发 `t_auth_refresh_session`，因此账号清理不能只修单个失败用例。
- `ProductionEmptyContextTest` 激活真实 `prod` profile 后，测试配置未同步 `ProductionSafetyGuard` 新增的微信小程序必填项，并且依赖了只在 `dev/test` 注册的企业核验、电子签章 mock 实现，导致目标断言执行前 Spring 上下文连续启动失败。
- 泛化原因：测试夹具和断言停留在早期迁移状态，没有随最终数据库依赖图和种子语义同步演进。
- 后续检查：新增账号外键后检索所有 `DELETE FROM t_account`，并按登录行为核对会话、凭证和审计子表；一个聚合的子表同时引用根、附件或明细时，按实际外键依赖从叶子表到根表清理，不假定数据库会为多个级联分支自动选择安全顺序；新增或复用固定种子 ID 后检索所有按该 ID、路由或权限做出的测试断言。
- 生产 profile 上下文测试必须同步 `ProductionSafetyGuard` 的必填配置，并在测试类中显式 `@MockBean` 外部服务端口；不得为通过测试而削弱生产安全校验，也不得把仅供测试的 mock provider 注册进生产 profile。
