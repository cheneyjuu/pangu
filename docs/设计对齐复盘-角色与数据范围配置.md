# 角色与数据范围配置设计对齐复盘

## 问题

`角色与数据范围配置` 页面连续出现设计对齐偏差：

- `新建角色` 被放在页面级 `PageHeader`，导致切到 `用户角色分配` tab 后仍然出现；但原设计里该动作属于 `权限矩阵配置`。
- `用户角色分配` 被实现成“用户列表 + 右侧分配面板”，偏离原设计的“全宽用户列表 + 变更角色弹窗”。
- 额外暴露 `用户账号管理` 菜单后，与 `用户角色分配` 在“新增用户 / 新增工作身份 / 分配角色”上形成重复入口。

## 原因

实现时优先按后端 `自然人账号(t_account) + 工作身份(sys_user)` 模型补能力，没有先把原设计当作页面布局契约逐项核对，导致把“可实现的功能面板”误当成“设计要求的交互结构”。

## 后续检查

类似管理台设计对齐任务，编码前必须先核对三件事：

1. 页面级动作是否应该跟随 tab 切换，还是属于全局页面。
2. 原设计中的入口是否已经覆盖现有独立菜单，避免重复入口。
3. 如果设计语义是“变更角色”，但后端只有“新增工作身份”，必须先确认业务口径，不能用前端文案掩盖模型差异。
