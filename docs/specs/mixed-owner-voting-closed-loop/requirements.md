# 纸质与线上并行表决闭环需求

> 业务关联：本小区有效议事规则明确允许时，在同一表决期向全体有权业主同时开放纸票和线上实名表决。
>
> 状态：重复票策略扩展实施中（2026-07-20）
>
> 上位业务规范：[维修与业主大会表决业务规范](../../business/维修与业主大会表决业务规范.md)
>
> 对应改造切片：[维修与表决能力改造计划 - 切片六](../../plans/维修与表决能力改造计划.md#切片六纸质与线上并行)

## 1. 范围

本切片实现“纸质与线上并行”这一独立的单次实际表决方式。它不是互联网表决中的个别纸质协助，也不是把两场投票的汇总结果相加。两个渠道必须共用同一表决包、冻结名册、时间窗口、规则快照和有效票唯一约束。

平台只有在本小区当前有效规则明确允许线上线下结合，且重复票规则已经确认并属于平台可执行策略时，才能建立该表决方式。采集方式与重复票策略是两个独立维度，不能因为平台先实现了“首张有效票优先”就把它写成所有小区的业务规则。

## 2. 功能需求

### R1 规则和单次方式

1. 有效规则必须同时允许线上线下结合和纸质、线上两个渠道；
2. 本次实际方式由有权自治主体在规则允许范围内确认，物业不能用默认开关启用；
3. 表决包锁定后，两个渠道共享同一材料、事项、名册、起止时间和计票口径；
4. 规则未明确允许或重复票处理依据未确认时，正式办理必须停止。

### R2 两个渠道同时开放

1. 冻结名册中的每个专有部分从投票开始即可选择线上或纸质办理，不要求先申请纸质协助；
2. 线上渠道继续执行 L2 实名、阅读确认和整包提交；业委会选举线上投票仍单独要求 L3 活体实名；
3. 纸质渠道继续执行送达核对、原件回收、一人录入和另一人复核；
4. 两个渠道都只能在同一表决时间窗口内形成有效票。

### R3 跨渠道唯一和规则裁决

1. 每个“表决事项 + 冻结名册项”只能有一张当前有效票；
2. `FIRST_VALID_WINS` 保留先形成的有效票，后到材料记录为重复；
3. `ONLINE_PREVAILS` 允许线上票替代既有纸质有效票，后到纸票不得覆盖线上票；
4. `PAPER_PREVAILS` 允许复核通过的纸质票替代既有线上有效票，后到线上票不得覆盖纸票；
5. 被替代票和后到材料均保留原件、裁决状态、替代关系和审计；
6. 并发由数据库当前有效票唯一约束兜底，最终最多形成一组当前有效票；
7. 物业不得临时选择策略，系统必须使用本次冻结规则。

### R4 进度和隐私

1. 管理端分别展示线上已完成人数、纸票送达、回收、待复核和重复材料数量；
2. 管理端不得展示在线票面选择；纸票选择只向有纸票录入复核权限的工作人员开放；
3. 业主端只展示本人每个专有部分是否已完成及回执，不回显具体选择；
4. 重复材料使用“重复材料，未重复计票”等业务用语，不显示数据库冲突或异常堆栈。

## 3. 验收场景

1. 有效规则不允许并行时，不能创建纸质与线上并行会次；
2. 同一场表决中，甲在线提交、乙通过双人复核纸票，结果均进入同一有效票台账；
3. `FIRST_VALID_WINS` 下两个到达顺序均保留先形成票；
4. `ONLINE_PREVAILS` 下纸票先、线上票后时，线上票成为当前有效票且纸票保留被替代记录；
5. `PAPER_PREVAILS` 下线上票先、纸票后时，纸票成为当前有效票且线上票保留被替代记录；
6. 两个渠道并发提交时，每个事项最多一张当前有效票，结果符合冻结策略；
7. 业主和普通物业人员均看不到其他业主的线上选择。

## 4. 非本切片范围

- 互联网表决中仅为个别业主提供纸质协助；
- 本地规则约定允许投票期内主动撤回、反复改票或使用三种既定优先策略以外的冲突策略；
- 从历史汇总数反推逐户、逐事项或渠道来源；
- 没有有效本地规则时用平台默认值启用混合方式。
