# 纸质与线上并行表决闭环需求

> 业务关联：本小区有效议事规则明确允许时，在同一表决期向全体有权业主同时开放纸票和线上实名表决。
>
> 状态：实施中（2026-07-19）
>
> 上位业务规范：[维修与业主大会表决业务规范](../../business/维修与业主大会表决业务规范.md)
>
> 对应改造切片：[维修与表决能力改造计划 - 切片六](../../plans/维修与表决能力改造计划.md#切片六纸质与线上并行)

## 1. 范围

本切片实现“纸质与线上并行”这一独立的单次实际表决方式。它不是互联网表决中的个别纸质协助，也不是把两场投票的汇总结果相加。两个渠道必须共用同一表决包、冻结名册、时间窗口、规则快照和有效票唯一约束。

平台只有在本小区当前有效规则明确允许线上线下结合，且重复票规则已经确认采用“先形成的有效票计入”时，才能建立该表决方式。

## 2. 功能需求

### R1 规则和单次方式

1. 有效规则必须同时允许线上线下结合和纸质、线上两个渠道；
2. 本次实际方式由有权自治主体在规则允许范围内确认，物业不能用默认开关启用；
3. 表决包锁定后，两个渠道共享同一材料、事项、名册、起止时间和计票口径；
4. 规则未明确允许或重复票处理依据未确认时，正式办理必须停止。

### R2 两个渠道同时开放

1. 冻结名册中的每个专有部分从投票开始即可选择线上或纸质办理，不要求先申请纸质协助；
2. 线上渠道继续执行活体实名、阅读确认和整包提交；
3. 纸质渠道继续执行送达核对、原件回收、一人录入和另一人复核；
4. 两个渠道都只能在同一表决时间窗口内形成有效票。

### R3 跨渠道唯一和先到有效

1. 每个“表决事项 + 冻结名册项”只能有一张有效票；
2. 先完成线上整包提交后，后到纸票仍保存原件、录入和复核记录，但逐事项标记为重复，不覆盖线上票；
3. 先完成纸票复核后，线上提交被拒绝并形成不含票面选择的冲突审计；
4. 并发由数据库唯一约束兜底，最终最多形成一组有效票；
5. 物业不得临时选择纸票优先或线上优先。

### R4 进度和隐私

1. 管理端分别展示线上已完成人数、纸票送达、回收、待复核和重复材料数量；
2. 管理端不得展示在线票面选择；纸票选择只向有纸票录入复核权限的工作人员开放；
3. 业主端只展示本人每个专有部分是否已完成及回执，不回显具体选择；
4. 重复材料使用“重复材料，未重复计票”等业务用语，不显示数据库冲突或异常堆栈。

## 3. 验收场景

1. 有效规则不允许并行时，不能创建纸质与线上并行会次；
2. 同一场表决中，甲在线提交、乙通过双人复核纸票，结果均进入同一有效票台账；
3. 甲在线提交后又收到其纸票，纸票原件完整保存，各事项标记为重复且不重复计票；
4. 乙纸票先完成复核后再尝试线上提交，线上请求被拒绝并留下冲突审计；
5. 两个渠道并发提交时，每个事项最多一张有效票，且后到材料不覆盖先形成的有效票；
6. 业主和普通物业人员均看不到线上选择。

## 4. 非本切片范围

- 互联网表决中仅为个别业主提供纸质协助；
- 本地规则约定“纸票优先”“线上优先”或允许改票的其他冲突策略；
- 从历史汇总数反推逐户、逐事项或渠道来源；
- 没有有效本地规则时用平台默认值启用混合方式。
