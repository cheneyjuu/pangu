# 纸质与线上并行表决闭环任务

> 业务关联：跟踪混合渠道的规则守卫、跨渠道唯一、冲突留痕和三端接入。
>
> 状态：实施中（2026-07-19）

## 1. 规则和实际方式

- [x] 仅允许有效规则明确配置 `ONLINE_AND_OFFLINE + PAPER_AND_ONLINE`；
- [x] 要求已确认的 `FIRST_VALID_WINS` 重复票处理规则；
- [x] 冻结为统一表决包 `PAPER_AND_ONLINE`，不建立第二套名册或计票结果。

## 2. 渠道和冲突

- [x] 混合方式直接开放纸票和线上入口，不要求纸质协助申请；
- [x] 线上先形成时，后到纸票保存原件并标记逐事项重复；
- [x] 纸票先形成时，后到线上请求拒绝并写不含选择的冲突审计；
- [x] 验证并发下数据库唯一约束和线上整包回滚。

## 3. 投影和接端

- [x] 统一管理办理台展示渠道进度、重复材料和冲突数量，不返回线上选择；
- [ ] 神农业主端按专有部分展示可选渠道、完成状态和本人回执；
- [ ] 盘古/瑶池仅在有效规则允许范围内选择本次方式，并提供纸票办理台。
