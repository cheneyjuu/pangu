# 纸质与线上并行表决闭环设计

> 业务关联：让两个采集渠道在同一冻结名册上竞争同一有效票位置，并完整保留后到纸质原件和线上冲突审计。
>
> 状态：重复票策略扩展实施中（2026-07-20）
>
> 需求依据：[requirements.md](requirements.md)

## 1. 设计原则

```text
同一表决包 + 同一冻结名册 + 同一时间窗
           |
      +----+----+
      |         |
   线上整包   纸票双人复核
      |         |
      +----+----+
           |
  表决事项 + 名册项唯一有效票
```

`PAPER_AND_ONLINE` 只控制采集入口，不建立第二份票或第二套计票结果。哪张重复票保持有效由冻结的 `DuplicateVotePolicy` 决定，不由渠道适配器或物业临时决定。

## 2. 规则与表决包

- `ONLINE_AND_OFFLINE + PAPER_AND_ONLINE` 与任一已支持的重复票策略映射为统一表决包 `CollectionMode.PAPER_AND_ONLINE`；
- 冻结摘要包含规则版本、实际方式、全部事项、材料、模板、时间窗口和名册摘要；
- 纸质送达无需纸质协助申请；非选举线上表决仍须本人阅读确认和 L2 实名，业委会选举线上投票才要求 L3；
- 表决开始后不得切换为互联网主渠道或纯纸质方式。

## 3. 跨渠道裁决

### 3.1 策略裁决

统一裁决服务接收既有票、新票和冻结策略，返回保留、替代或接收。渠道适配器只负责验证原始材料和身份，不自行决定优先级。

- 首张有效票优先：后到票记录为重复；
- 线上有效票优先：线上票可替代纸质票，纸质票不能替代线上票；
- 纸质有效票优先：纸质票可替代线上票，线上票不能替代纸质票。

被替代票保留原票、替代票引用、策略、原因和时间。计票只查询当前有效票。

### 3.3 并发

统一有效票表以“表决事项 + 冻结名册项”的当前有效票唯一索引兜底。裁决和写入在同一事务完成；并发冲突后重新读取既有票并按同一冻结策略收敛，不能以数据库到达顺序替代业务规则。

## 4. 工作台投影

统一办理台返回：

- 纸票送达和复核状态；
- 回收纸票、录入版本和逐事项处理结果；
- 互联网表决中的纸质协助请求；
- 线上已完成的数量汇总和跨渠道冲突数量。

办理台不返回在线选择。业主端按每个专有部分返回渠道可用性、完成状态和本人回执，同样不回显选择。

## 5. 测试策略

- 规则测试验证三种已支持策略均可执行，未知或不完整策略在启用前阻断；
- 集成测试同时完成一户线上和一户纸票；
- 顺序冲突测试覆盖三种策略、两个到达顺序和幂等重试；
- 并发与数据库测试验证最多一张有效票；
- Web 投影测试验证管理端和业主端不泄露线上选择。
