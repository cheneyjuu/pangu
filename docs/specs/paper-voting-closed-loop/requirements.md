# 纸质投票闭环需求

> 业务关联：业主大会及后续维修相关业主共同决定中的纸质材料送达、纸票回收、双人录入复核和逐票留痕。
>
> 状态：实施中（2026-07-19）
>
> 上位业务规范：[维修与业主大会表决业务规范](../../business/维修与业主大会表决业务规范.md)
>
> 对应改造切片：[维修与表决能力改造计划 - 切片四](../../plans/维修与表决能力改造计划.md#切片四纸质投票闭环)

## 1. 范围

本切片在统一表决执行内核之上实现纸质渠道适配器。纸质原件、经办记录和复核过程独立留痕；只有经过核对的有效送达和经过两名不同人员一致录入的有效纸票，才能写入统一送达及有效票台账。

本切片不把物业录入解释为代业主投票，不根据附件名称或上传成功自动确认送达、选票有效性或业主选择，也不补造历史纸票的逐户事实。

## 2. 业务术语

- **纸质表决票模板**：本次表决安排中已经归档并锁定的正式空白票原件、版本和摘要；系统不自行生成用印结论。
- **送达登记**：经办人员登记向冻结名册中的专有部分送达表决材料的方式、接收人、时间和证据。
- **纸票回收登记**：登记实际收到的纸票编号、对应专有部分、回收时间及原件。
- **录入**：经办人员依据纸票原件录入每个表决事项的票面意思；提交后的录入版本不可直接修改。
- **复核**：另一名人员对照原件核对录入内容，确认或退回并说明差异。
- **重复材料**：对应事项和专有部分已有有效票后收到的纸票；保留原件和原因，但不覆盖有效票。

## 3. 功能需求

### R1 锁定正式纸票模板

1. 本次表决实际允许纸质渠道时，正式办理前必须绑定一份 `PAPER_BALLOT_TEMPLATE` 原件；
2. 表决包锁定模板材料编号、内容摘要和版本，之后不得替换；
3. 冻结规则要求纸质用印时，经办方必须确认上传的是已经按规则完成用印的正式模板；系统只记录确认和原件，不自动判断印章真伪；
4. 纸票回收时必须引用本次锁定模板，旧版本或其他会次模板不能形成有效票。

### R2 送达先登记后核对

1. 送达登记必须绑定表决包、冻结名册中的专有部分、接收人、规则允许的送达方式、送达时间、原始证据和经办人；
2. 登记后处于待核对状态，不能立即成为统一内核的有效送达；
3. 核对人确认方式、时间、接收人和证据符合本次规则后，系统才写入统一送达台账；
4. 核对不通过必须保存原因，可补充新的送达登记，但不能覆盖原记录；
5. 同一专有部分同一纸质渠道只能有一条生效送达，后续材料作为补充证据留存。

### R3 回收纸票原件

1. 纸票回收必须保存本次表决包内唯一的表决票编号、对应专有部分、模板摘要、回收时间、经办人和原件材料；
2. 原件必须是当前会次已上传且类型为 `PAPER_BALLOT` 的受控材料；
3. 上传或登记原件不等于纸票有效，也不产生任何计票记录；
4. 回收登记不得修改或删除，录入错误通过作废原因和重新登记保留轨迹。

### R4 录入与第二人复核

1. 录入人员对同一张纸票内的全部表决事项分别记录“有效选择”或“无效并说明原因”；
2. 复核必须由不同的当前有效系统用户完成，复核人对照原件确认或退回整份录入；
3. 复核确认后，有效选择写入统一有效票台账，无效票只进入纸票审计台账；
4. 复核退回时不得修改或删除原录入，必须保存差异说明；录入人员提交新的修订版本后重新由不同人员复核；
5. 未复核、被退回或修订中的纸票不产生有效票。

### R5 无效、重复和跨渠道唯一

1. 无效票必须保存结构化原因和必要的补充说明，不得伪造为弃权；
2. 对同一表决事项和专有部分，已有线上或纸质有效票时，后收到的纸票标记为重复材料并保存冲突指向；
3. 重复材料不能覆盖、撤回或修改先形成的有效票；
4. 并发复核仍由统一有效票数据库唯一约束兜底；
5. 本切片只实现“先形成的有效票保留”的安全基线；需要其他优先规则时，必须由已确认结构化议事规则及后续混合投票切片实现。

### R6 权限、隐私与审计

1. 纸质送达、回收和录入仅向具有当前办理权限的工作身份开放，租户、会次和材料归属由后端校验；
2. 录入人、复核人、处理人、时间、原件摘要和状态变化全部保留审计；
3. 业主端只可查看本人名下专有部分的送达状态、是否已经参与和公开材料，不可查看本人或他人的具体选择；
4. 管理端不得以技术状态码代替业务语言，应展示“待核对送达”“待第二人录入”“录入不一致”“已计入”“无效”“重复材料”等状态。

## 4. 验收场景

1. 经办人登记纸质送达后，统一送达台账仍为空；核对通过后才产生有效送达记录；
2. 同一用户不能既录入又复核同一份纸票录入；
3. 另一名用户复核确认有效选择后，统一有效票台账恰好新增一票；
4. 复核确认某事项无效时，不新增有效票，并可查到无效原因和原件；
5. 复核退回后不新增有效票，原录入和差异说明保留，新修订需重新复核；
6. 某专有部分已经线上投票后，其纸票复核完成时保留为重复材料，线上有效票不变；
7. 旧模板、其他会次材料、非冻结名册房屋、已截止表决和越权账号均被后端拒绝；
8. 业主查询本人参与状态时看不到任何选项内容或其他业主信息。

## 5. 非本切片范围

- 线上实名投票页面、电子确认和纸质补充申请；
- 面向所有业主同时开放纸质与线上时的渠道切换和冲突处理工作台；
- 表决结果公告、异议处理和维修工程结果推进；
- 从历史汇总、截图或台账数字反推单户送达和纸票。
