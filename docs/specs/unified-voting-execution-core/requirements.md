# 统一表决执行内核需求

> 业务关联：为业主大会、相关业主共同决定和后续维修决定提供同一套表决包、名册快照、送达、有效票、计票和审计事实。
>
> 状态：已完成（2026-07-19）
>
> 上位业务规范：[维修与业主大会表决业务规范](../../business/维修与业主大会表决业务规范.md)
>
> 对应改造切片：[维修与表决能力改造计划 - 切片三](../../plans/维修与表决能力改造计划.md#切片三统一表决执行内核)

## 1. 范围

本切片建立可被不同上层业务复用的表决执行内核，并先将现有业主大会纸质表决接入。纸票双人复核、业主端线上实名投票和混合渠道冲突工作台分别由后续切片完成。

本切片不以新建一套平行票据表为目标。现有通用议题、有效票和计票引擎继续复用，业主大会专用的表决包、送达和票据留痕向通用内核收敛。

## 2. 领域语义

- **上层业务**：决定谁可以发起、本次决定范围、通过后如何推进业务；不自行保存另一套有效票。
- **表决包**：一次表决中不可拆分的方案版本、规则依据、决定范围、实际收集方式和时间窗口。
- **表决事项**：表决包内独立计票的问题；每个事项恰好属于一个表决包。
- **表决人名册快照**：表决包锁定时确定的专有部分、面积、表决权代表和计票基数，后续不回读实时房屋名册。
- **有效票台账**：纸质和线上渠道共享的唯一计票事实，不是两个渠道各自汇总后相加。

## 3. 功能需求

### R1 表决包必须绑定不可变依据

1. 表决包必须保存租户、业务类型及关联编号、方案版本及摘要、规则快照及摘要、决定范围、本次实际收集方式和投票时间窗口；
2. 缺少任一依据的新表决包不得锁定；
3. 锁定后不得修改方案、规则、范围、收集方式或时间窗口。

### R2 锁定时冻结表决人名册

1. 冻结快照以“专有部分”为基本行，保存当时的楼栋、面积、表决权代表自然人及对应业主房屋身份；
2. 专有部分面积只能计入一次，人数按当次快照的合法计票口径去重；
3. 投票和计票只能使用该快照，快照后修改房屋或代表关系不能改写历史表决；
4. 系统不得以欠费、账号冻结或一般系统状态自行排除业主的表决基数；任何排除都必须有明确的业务依据和审计记录。

### R3 送达与投票分离

1. 送达记录绑定表决包、专有部分、接收人、方式、时间、原始证据和复核状态；
2. 送达是后续未反馈处理的依据，不能用“已投票”或一段自由文本替代；
3. 具体原始证据和复核工作台由纸质投票切片继续完成。

### R4 不同渠道共享同一有效票唯一性

1. 一般或重大决议以“表决包 + 表决事项 + 专有部分”作为有效票唯一键；
2. 纸质票和线上票不得同时成为有效票，并发由数据库唯一约束兜底；
3. 有效票保存快照中的自然人、专有部分、面积、渠道、选择、时间和与表决包的内容摘要；
4. 后到的重复票不覆盖已有有效票。对重复原件的存证和冲突处理在混合投票切片完成。

### R5 计票和结果必须可追溯

1. 计票只读取表决包冻结的计票基数、有效票和规则快照；
2. 结果必须保存表决包、表决事项、名册快照、方案摘要和规则摘要；
3. 已结算结果不得因后续名册、规则或方案变更而重算为不同结论。

### R6 租户、状态和审计

1. 表决包、事项、名册、送达、票和结果必须属于同一租户；
2. 未锁定、未到投票时间、已截止或已作废的表决包不接受有效票；
3. 锁定、开始投票、收到有效票、截止、结算和作废动作均保留操作人、时间和当时摘要。

## 4. 验收场景

1. 业主大会的表决事项锁定后，可查到唯一通用表决包和名册快照；
2. 锁定后新增、删除或改变房屋代表关系，不会改变该表决包的名册和计票基数；
3. 不在冻结名册中的房屋或自然人不能形成有效票；
4. 同一表决事项的同一专有部分先形成线上有效票后，纸质票不能再成为有效票，反向同理；
5. 表决结果可追溯到表决包、方案摘要、规则摘要、名册摘要和每张有效票；
6. 历史通用议题或缺少真实快照的老表决不会被自动伪造为新内核表决包。

## 5. 非本切片范围

- 纸质表决票模板、回收和双人录入复核工作台；
- 业主端线上实名投票页面及签名确认；
- 纸质与线上并行时的重复票原件归档和冲突处理工作台；
- 维修项目接入统一表决包；
- 从历史汇总数据推导或补造单户送达、票面或表决权代表。
