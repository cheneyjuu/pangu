# 业主大会议事规则采用原件、结构化配置和执行快照共同治理

> 业务关联：业主大会筹备、会议形式、方案公示、会议通知、材料送达、纸质与线上投票、计票和结果公告。
>
> Status: accepted
>
> Date: 2026-07-18
>
> Revised: 2026-07-19

## Context

各小区实际生效的《业主大会议事规则》可能对会议形式、通知、公示、有效送达、未反馈处理、委托代理、纸质和线上投票、计票及结果公告作出不同约定。上海市示范文本不能替代江湾国际公寓实际生效的规则。

系统同时面临三类要求：

1. 保存具有证明力的规则原件；
2. 使用稳定的结构化规则驱动流程和校验；
3. 在单次会议开始后保持规则、方案和计票口径不可变。

仅保存文件会迫使运行时重复解析；仅保存可编辑配置无法证明字段来源；把上传文件直接设为生效规则则会把录入或解析错误提升为业务事实。

上海市 2026 年示范文本同时要求系统面对纸质、互联网以及纸质兜底场景，并禁止同一事项跨方式重复投票。因此“平台固定线上”或“平台固定纸质”都不能作为目标模型。

## Decision

### 1. 一个规则版本包含三类记录

每个规则版本同时保存：

1. **规则原件**：原始文件、摘要、生效日期、来源和备案信息；
2. **结构化配置**：允许的会议和表决方式、有效送达方式、未反馈处理、代理规则、纸质用印、线上身份要求、跨渠道重复票处理、计票和公告要求；
3. **确认与审计**：录入人、逐项核对人、确认依据、确认时间和变更原因。

每个结构化字段尽可能关联原件页码和条款。OCR 或模型只能预填草稿，不能自动产生生效配置。

### 2. 规则版本按状态启用

版本生命周期为：

`DRAFT -> PENDING_CONFIRMATION -> ACTIVE -> SUPERSEDED`

- 经办人员可以上传原件并录入草稿；
- 有权确认主体必须根据当前任期和真实授权核验，不能只依赖账号显示名称；
- 逐项确认完成后才能启用；
- `ACTIVE` 版本不可修改，变化必须新建版本；
- 物业服务企业不能单独使小区议事规则生效。

### 3. 规则允许方式与单次实际方式分离

规则版本记录本小区允许的方式。单次会议在允许范围内选择并冻结实际方式：

- `PAPER`：纸质投票；
- `ONLINE`：线上实名投票；
- `PAPER_AND_ONLINE`：纸质和线上并行。

表决方式与送达方式独立建模。一次会议可以电子通知并收纸质票，也可以按规则采用其他组合；系统不能用一个“渠道”字段同时表达两个概念。

### 4. 单次会议冻结执行快照

进入正式办理前，系统从当时有效规则生成不可变快照，并与表决包一起冻结：

- 表决事项和方案版本；
- 表决人名册及面积、人数基数；
- 实际表决方式和时间窗口；
- 有效送达方式及未反馈处理；
- 纸质、线上和重复票规则；
- 对应材料清单和摘要。

后续规则换版不影响在办或历史会议。没有有效规则版本时可以保存内部草稿，但不得进入正式送达、投票或计票。

### 5. 三种方式共用表决执行内核

纸质、线上和混合投票共用同一表决包、表决权名册、逐户送达、有效票和计票模型。

系统以“表决事项 + 专有部分”为一张有效票的唯一边界：

- 纸票和线上票不能对同一专有部分同时有效；
- 后收到的重复票保留证据和无效原因，不能覆盖原有效票；
- 并发提交由数据库唯一约束兜底；
- 本小区规则要求的特殊处理只有在形成已确认结构化配置后才能执行。

维修事项、业主大会或其他上层流程可以有不同的发起和结果推进规则，但不得分别维护互相矛盾的投票真相。

### 6. 未完整实现的能力必须阻断

某种规则或表决方式只有在身份、名册、送达、原始票、去重、计票和审计全部可验证后才能进入正式办理。

允许范围大于当前系统能力时，系统应明确说明尚未支持并阻断正式流程；不得把平台硬编码的纸质或线上方式冒充本小区规则，也不得只增加前端选项。

## Consequences

- 后端以已确认规则配置和单次快照作为运行时真相，原件用于审计及争议复核；
- 管理端必须区分录入、核对和启用，不能把上传成功显示为规则已生效；
- 社区设置展示本小区规则允许的方式，单次会议页面展示本次实际方式；
- 业主端根据本次实际方式显示线上入口、纸质办理状态或混合办理路径；
- 混合投票必须先具备跨渠道唯一票约束，不能事后简单相加两套统计；
- 江湾实际规则未取得前，只能建设通用登记和执行能力，不能用示范文本补造生效配置。

## Considered Alternatives

- **只上传文件、运行时自动解析**：拒绝。解析结果不稳定，无法保证多次执行一致。
- **只保存人工配置、不保存原件**：拒绝。无法证明字段来源并支持争议复核。
- **上传或解析后自动启用**：拒绝。技术操作不能替代有权主体确认。
- **平台固定线上或固定纸质**：拒绝。不符合不同小区规则和不同业主群体的真实办理需要。
- **纸质和线上各自计票后相加**：拒绝。无法可靠防止同一专有部分跨渠道重复投票。
