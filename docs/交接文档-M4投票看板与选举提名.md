### 2.1 选举提名「按手机号关联业主」（commit `2ccf321`，本会话主线）

**关键 SQL**（`OwnerPropertyMapper.xml`）：
```sql
SELECT cu.uid AS uid, a.phone AS phone,
       MIN(op.building_id) AS building_id, MIN(op.room_id) AS room_id
FROM t_account a
JOIN c_user cu ON cu.account_id = a.account_id
JOIN c_owner_property op ON op.uid = cu.uid AND op.tenant_id = #{tenantId}
WHERE a.phone LIKE CONCAT(#{phonePrefix}, '%')   -- 命中 idx_account_phone
GROUP BY cu.uid, a.phone
ORDER BY a.phone
LIMIT 20
```
> `phone` 作普通明文列映射，**绝不选 `real_name`**（SM4 加密列，见坑 6.2）。

## 4. 架构 & 约定速查（强约束，违反会被拒）

- **分层**：bootstrap → interfaces → application → domain ← infrastructure。`pangu-domain` framework-light（无 Spring/MyBatis，Lombok 极简）；domain 单例（如 `*VotingEngine`）在 `bootstrap/.../config/DomainConfig.java` 里 `@Bean` 装配，**不在 domain 模块加注解**。
- **Mapper** 必须在 `com.pangu.infrastructure.persistence.mapper` 包下，否则 `@MapperScan` 扫不到。
- **错误契约**：`throw new AppException(ErrorCode, ...)`，链 cause；新代码**禁用**废弃的 `(int code, String message)`。`GlobalExceptionHandler` 是唯一出口。
- **安全上下文**：controller/service 用 `SecurityUtils`（`pangu-interfaces`），**不直接** `SecurityContextHolder`。
- **多租户**：`TenantContext` 是 `ThreadLocal<Long>`；任何 set 的代码路径必须 `finally` 里 `clear()`。
- **SM4**：敏感列用 `Sm4EncryptTypeHandler`（mapper XML 里挂），不在 service 调 `Sm4Util`。
- **Flyway**：`pangu-bootstrap/.../db/migration/`，`V<ver>__<desc>.sql`；**不加 V0、不改历史版本**。
- **提交**：Conventional Commits 带模块/feature scope；**只在用户明确要求时**提交/推送；commit 尾 `Co-Authored-By: Claude <noreply@anthropic.com>`，PR 尾 `🤖 Generated with [Claude Code](https://claude.com/claude-code)`。
- **代码风格**：4 空格缩进；显式 import（无通配、无全限定名）；中文回答。

## 6. 已知坑（最高价值，务必先看）

**6.1 启动 spring-boot 的正确姿势** —— `mvn -pl pangu-bootstrap -am spring-boot:run` 在 reactor 根**解析不到 `spring-boot:` 前缀**（插件只在 bootstrap pom 声明），会 `NoPluginFoundForPrefixException`。正确做法：进 `pangu-bootstrap/` 目录跑全限定 goal：
```bash
cd /Users/juchen/Documents/workspace/pangu/pangu-bootstrap
nohup mvn org.springframework.boot:spring-boot-maven-plugin:run > /tmp/pangu-boot.log 2>&1 &
```

**6.2 SM4 `real_name` 解密会崩** —— `Sm4EncryptTypeHandler` 读时调 `Sm4Util.decryptHex`，对开发期 `MOCK_` 明文（非 hex）会**抛 `SM4 解密失败`**。所以任何检索 SQL **不要选 `real_name`**。`phone` 是 `t_account.phone`（明文 + UNIQUE + `idx_account_phone` 索引），是理想关联键。现有 `vote-details` 把姓名留作「数据缺口占位」也是这个原因。

**6.3 yaochi 无 tsconfig.json** —— LSP 会狂报 `'React' refers to a UMD global`（2686）——**全是误报**。判定构建是否通过**只看** `npm run build`（vite build），不看 LSP 诊断。

**6.4 zsh `UID` 是只读变量** —— 脚本里 `UID=$(...)` 会报 `failed to change user ID: operation not permitted`。换个变量名（如 `UID_RESOLVED`）。

**6.5 `grep --include=*.java` 在 zsh 下** 可能 `no matches found`——直接给目录路径，别用 `--include` glob。

**6.6 手机号前缀匹配的语义** —— `LIKE 'prefix%'` 是真前缀。测试账号 张三=`13800000012`、李四=`13800000113`：`phone=138000001` 只命中李四（其第 9 位是 `1`），不命中张三（第 9 位 `0`）——这是正确行为，别误判为 bug。
