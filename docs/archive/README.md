# 历史文档归档

> 业务关联：保存过去的需求、方案、路线图和实施记录，避免与当前业务规范混用。
>
> 状态：历史归档，非业务依据

本目录中的文档可能包含过时、互相矛盾或未经正式材料验证的结论。保留它们是为了追溯历史，不表示内容仍然正确。

- `business/`：旧 PRD、旧业务方案和旧权限说明；
- `plans/`：历史路线图、修复计划和设计提示；
- `adr/`：待复核或已废止的架构决策；
- `implementation-history/`：特定版本的实施总结、交接和复盘。

当前维修与表决业务统一以 [维修与业主大会表决业务规范](../business/维修与业主大会表决业务规范.md) 为准。

引用归档文档中的事实前，必须重新核对原始材料，并把确认后的结论写入当前权威业务规范，而不是恢复归档文档的权威状态。
