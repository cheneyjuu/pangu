# [已废止] 物业模式切换走完整双 2/3 议案 + 党组前置审查 + 90 天公示

> Status: superseded
> Date: 2026-06-19

> 废止说明：原文把参与门槛、通过门槛、党组织审查和公示期限拼成了平台统一规则，缺乏足够依据。物业服务方式或合同事项应先识别具体决定事项，再按《民法典》第二百七十八条、地方规定和本小区已生效议事规则分别确定参与及通过门槛；党组织、街道和公示环节也只能在存在适用依据时进入流程。当前业务口径见 [维修与业主大会表决业务规范](../../business/维修与业主大会表决业务规范.md)。下文仅保留为历史记录，不得据此实现。

## Context

业主小区在生命周期中**会切换物业模式**：从包干制（LUMP_SUM）到筹金制（FUND_RAISING）以追求账目透明，从筹金制到信托制（TRUST）以追求双签 + 司法链上链，从信托制降级到筹金制以避免每笔上链费用。模式切换的影响范围远大于单一议案——它涉及（a）物业公司商业利益重大调整、（b）业主资金账户性质变更、（c）数据范围权限重定义、（d）可能引发物业公司挤兑维修资金跑路。

四种潜在门槛：

1. **业委会议决议即可**：成本最低，但业委会越权代表全体业主决定模式；
2. **简单多数议案**：与一般经营合同同级；
3. **完整双 2/3 议案 + 街道办前置审查 + 党组审查 + 90 天公示**：最高规格；
4. **街道办指定**：政府越级。

## Decision

所有模式切换走**完整双 2/3 议案**（专有面积 ≥ 2/3 AND 人数 ≥ 2/3，与 [CONTEXT.md - Q18 选聘 / 解聘物业议案] 同级），叠加：

- **街道办前置审查** + **党组前置审查**（与 [Q23.4 重大资金党组审查] 同源）；
- **90 天公示期**（让业主、物业、外部审计 / 担保方都有充足时间提异议或退出）；
- 通过后进 `IN_TRANSITION`（默认 60 天，最长 120 天）走第三方审计 + 资金清算（[CONTEXT.md - 模式切换资金清算 + 第三方审计]）；
- 切换议案落 `t_property_mode_switch_proposal`，走与 [CONTEXT.md - 切换失败 4 种回滚触发] 一致的回滚机制。

## Consequences

- **正向**：双 2/3 + 党组三重门槛防止业委会主任与新物业勾结私吞维修资金；90 天公示给外部审计 / 担保方退出窗口；IN_TRANSITION 60-120 天硬上限防止"切换中无限拖着"；超期触发自动回滚（[切换失败 4 种回滚触发 - TIMEOUT_EXPIRED]）。
- **反向**：从决定切换到完成切换需 **6-9 个月**（议案 90 天公示 + 60-120 天 IN_TRANSITION），在此期间小区运营受 [模式切换 IN_TRANSITION 分级冻结] 约束（不能起新议案 / 新经营合同 / 动用维修资金作非紧急支出）；高门槛意味着模式切换是"重大不可逆决策"，业主必须在切换前充分调研——这是有意为之的设计（防止冲动切换），不是缺陷。

## Considered Alternatives

- **业委会议决议即可**：被拒。业委会代表全体业主重大决策违反 [民法典 §278] 业主大会保留权限；业委会主任可与新物业勾结。
- **简单多数议案**：被拒。模式切换涉及业主资金账户性质变更，必须保留与"选聘 / 解聘物业"同级的双 2/3 门槛——否则少数派业主的资金权益无法保障。
- **街道办指定**：被拒。政府越级代表业主决定属于行政干预，违反《物业管理条例》§11 业主大会权能。
