# 维修工程创建与参考询价修复计划

日期：2026-07-19
范围：物业端维修工程项目创建、参考询价入口与供应商询价面板。

## 1. 已确认事实

1. 项目台账的“参考询价”入口会同时读取项目详情和询价资料；现有实现将两项请求绑定为同一结果。询价资料读取失败时，项目详情也不会落入页面状态，导致抽屉表现为空白或持续加载。
2. 询价面板对询价资料请求失败只清空本地数据，没有区分“正在读取”和“读取失败”，物业人员无法知道失败原因，也没有明确重试入口。
3. 创建项目的第二步把项目级 `budgetTotal` 标为“送审/决定预算（含税）”。后端把它作为方案预算和后续资金、授权校验的输入，并不等同于已经由业主决定或授权的最终金额。
4. 当前“维修点位与维修对象”前端把“空间名称、构件、具体部位”一并设为必填；后端真实必填项只有维修对象名称、位置归属、现场问题、原因状态、拟定措施，以及原因已确认时的原因依据。前端要求比业务和服务端契约更严。
5. 当前“参照房屋”是内部位置类型 `REFERENCE_ROOM` 的直接翻译。它只用于把点位关联到房屋档案以便定位和校验，不决定共用性质、费用承担、表决范围或资金归属。

## 2. 资料核对与业务结论

本计划已对照 `docs/sample` 中的设备门、排水管、绿化带水管和地库排水泵维修报价单。

| 信息类别 | 建项阶段来源 | 是否每个项目都填写 | 进入哪一环节 |
| --- | --- | --- | --- |
| 维修对象、现场问题、拟定措施 | 报修工单、现场勘验、照片、维修建议 | 是 | 建项 |
| 楼栋、单元、关联房屋或公共部位 | 房屋/资产档案与现场定位 | 按位置类型填写 | 建项 |
| 原因依据 | 勘验记录、检测报告、现场照片 | 仅原因已确认时 | 建项 |
| 工程量、单位、初步估算与依据 | 已完成勘验、历史结算或可核验测算 | 有可靠依据时填写 | 建项 |
| 材料型号、品牌、人工拆分、单价、税率、报价总额 | 供应商报价或招标清单 | 不属于统一建项表单 | 询价/报价 |

样本显示，设备门、排水管、泵房设备和绿化水管的技术明细差异很大，不能为了统一表单而提前强制填写一组通用工程清单字段。建项只记录可核验的范围和问题；报价阶段再保留专业清单及含税报价。

## 3. 修复目标与方案

### 3.1 参考询价入口

1. 项目详情与询价资料分开读取：项目详情成功后立即渲染抽屉，询价资料失败不再阻断项目页面。
2. 询价资料读取失败时明确展示失败原因和“重新读取”操作，不以无限加载或空白替代错误。
3. 询价资料成功、失败、刷新均保留项目主体和既有办理记录，避免一次外围接口故障造成整页不可用。

### 3.2 初步预算语义

1. 将“送审/决定预算（含税）”改为“项目初步预算（含税）”。
2. 在字段旁说明：该金额用于建项、方案比较和后续送审准备，不是已决定、已授权或最终合同金额。
3. 决定或授权后的预算只能由后续有效决定、资金核验和锁定快照固化；创建草稿不能替代该流程。

### 3.3 维修点位表单

1. 将“参照房屋”改为“关联房屋（用于定位）”，位置类型改为“关联房屋位置 / 公共部位”。
2. 明确提示关联房屋只服务于位置定位，不自动决定费用承担或共用范围。
3. 用必填标记和就地说明区分三类字段：
   - 所有点位必须填写：维修对象名称、位置类别及对应位置、现场问题、原因状态、拟定措施；
   - 条件必填：原因已确认时填写原因依据；填写数量时填写单位；填写初步估算时填写估算依据；
   - 可选补充：空间/朝向、构件、具体部位、技术要求、关联报修事项。
4. 将定位、现场问题与措施、范围量与初步估算分组展示，移除前端对空间、构件、具体部位的伪必填校验。
5. 仅在建项阶段收集业务范围和可核验证据；材料规格、分项单价、税率等留在供应商报价阶段，不新增泛化字段或数据库迁移。

## 4. 非本次范围

1. 不根据单个工程的填写内容自动判断共用性质、承担房屋、资金账户或表决范围；这些结论必须在后续决定、资金核验和锁定流程中形成。
2. 不引入跨楼栋合并建项。
3. 不把供应商报价单的专业工程清单提前固化为所有维修项目的强制字段。
4. 业主端已有“关联房屋位置”的表达，本次不改动业主端数据契约。

## 5. 验收标准

1. 点击任一项目“参考询价”后，即使询价接口失败，也能看到项目标题、方案和明确的错误/重试状态，不出现白屏。
2. 新建项目第二步显示“项目初步预算（含税）”及其用途说明，不再暗示草稿阶段已作出决定。
3. 新建项目第三步显示必填标记；不填写空间、构件、具体部位仍可在其他必填信息完整时进入下一步。
4. 选择“关联房屋位置”时才要求选择关联房屋；选择“公共部位”时才要求填写公共部位名称。
5. 成功构建前端，并通过维修项目/询价后端流程测试；部署后完成生产可达性检查。
